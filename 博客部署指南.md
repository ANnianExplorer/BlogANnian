# Valaxy + Sakura 主题博客完整搭建指南

## 第一章：环境安装与项目初始化

### 1.1 Node.js 环境准备
```bash
# 推荐使用 nvm 管理 Node.js 版本
# Windows: https://github.com/coreybutler/nvm-windows
# macOS/Linux: https://github.com/nvm-sh/nvm

nvm install 20
nvm use 20

# 验证安装
node -v  # 应显示 v20.x.x
npm -v   # 应显示 10.x.x
```

### 1.2 创建项目
```bash
# 创建项目目录
mkdir blog && cd blog

# 初始化 valaxy（交互式选择 Sakura 主题）
npm create valaxy@latest

# 或者使用模板
npm create valaxy@latest my-blog -- --template sakura

# 进入项目目录
cd my-blog
```

### 1.3 安装依赖
```bash
# 安装 valaxy 框架
npm install valaxy

# 安装 Sakura 主题
npm install valaxy-theme-sakura

# 安装访问统计插件（可选）
npm install valaxy-addon-vercount

# 安装 Markdown 样式
npm install star-markdown-css

# 安装 TypeScript
npm install -D typescript

# 或者一次性安装所有依赖
npm install
```

import { ref, onMounted, onUnmounted, computed } from 'vue'
import { useSiteConfig, usePostList } from 'valaxy'

const siteConfig = useSiteConfig()
const postList = usePostList()
const siteSince = '2025-01-01'

const postCount = computed(() => postList.value?.length || 0)

const currentTime = ref('')
const runTime = ref('')
const siteStartTime = ref('')

function pad(n) {
  return n < 10 ? '0' + n : '' + n
}

function formatDateTime(d) {
  const y = d.getFullYear()
  const m = pad(d.getMonth() + 1)
  const day = pad(d.getDate())
  const h = pad(d.getHours())
  const min = pad(d.getMinutes())
  const s = pad(d.getSeconds())
  return y + '年' + m + '月' + day + '日 ' + h + ':' + min + ':' + s
}

function updateTime() {
  const now = new Date()
  const start = new Date(siteSince)
  const diff = now.getTime() - start.getTime()
  const days = Math.floor(diff / 86400000)
  const hours = Math.floor((diff % 86400000) / 3600000)
  runTime.value = days + '天' + hours + '时'
  currentTime.value = formatDateTime(now)
  siteStartTime.value = formatDateTime(start)
}

let timer

onMounted(() => {
  updateTime()
  timer = setInterval(updateTime, 1000)
})

onUnmounted(() => {
  if (timer) clearInterval(timer)
})
</script>

## 第五章：GitHub Pages 部署

### 5.1 创建 GitHub 仓库
1. 登录 GitHub
2. 创建新仓库（建议命名为 `BlogANnian`）
3. 不要勾选 "Initialize with README"

### 5.2 本地初始化并推送
```bash
# 进入项目目录
cd blog

# 初始化 git
git init
git add -A
git commit -m "init blog"

# 添加远程仓库
git remote add origin https://github.com/ANnianExplorer/BlogANnian.git

# 推送
git push -u origin main
```

### 5.3 GitHub Actions 配置
在 `.github/workflows/gh-pages.yml` 中配置：
```yaml
name: GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Pages
        uses: actions/configure-pages@v5

      - name: Use Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install Dependencies
        run: npm install

      - name: Build Valaxy Blog
        run: npm run build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

### 5.4 启用 GitHub Pages
1. 进入仓库 **Settings → Pages**
2. **Build and deployment → Source** 设为 **GitHub Actions**
3. 等待 workflow 运行完成（约 1-2 分钟）

### 5.5 访问网站
- 项目仓库：`https://ANnianExplorer.github.io/BlogANnian/`

## 第六章：错误解决汇总

### 6.1 图片路径错误
**错误信息**：
```
[UNRESOLVED_IMPORT] Error: Could not resolve '/BlogANnian/images/xxx.png'
```

**原因**：Valaxy 把 `/BlogANnian/` 前缀的路径当作模块导入

**解决**：使用 `/images/` 前缀（不带 `/BlogANnian/`）
```markdown
![图片](/images/image-xxx.png)  <!-- 正确 -->
![图片](/BlogANnian/images/image-xxx.png)  <!-- 错误 -->
```

### 6.2 缺少 frontmatter
**错误信息**：
```
[Vue warn]: Property "title" was accessed during render but is not defined on instance.
```

**原因**：文章缺少 frontmatter 的 title 字段

**解决**：添加完整 frontmatter
```yaml
---
title: 文章标题
date: 2025-10-18
tags:
  - 标签1
  - 标签2
categories:
  - 分类
cover: /images/cover/xxx.jpg
---
```

### 6.3 pnpm vs npm
**错误信息**：
```
Dependencies lock file is not found in /home/runner/work/.../BlogANnian.
Supported file patterns: pnpm-lock.yaml
```

**原因**：workflow 使用了 pnpm，但项目使用 npm

**解决**：在 workflow 中改用 npm
```yaml
- name: Install Dependencies
  run: npm install
```

### 6.4 图片被当作模块导入
**错误信息**：
```
import _imports_0 from '/BlogANnian/images/image-xxx.png'
```

**原因**：Valaxy 解析 Markdown 时把图片当作动态导入

**解决**：
1. 确保图片在 `public/images/` 目录
2. 使用 `/images/` 路径
3. 不要使用 `/BlogANnian/` 前缀

### 6.5 本地运行正常但 GitHub 构建失败
**排查步骤**：
1. 本地先运行 `npm run build` 测试
2. 检查图片路径是否正确
3. 检查 frontmatter 是否完整
4. 检查是否有乱码或特殊字符

检查图片路径、frontmatter、是否有乱码

1. 验证 Settings → Pages → Source 设为 GitHub Actions
2. 检查 workflow 是否运行成功

1. 检查图片是否在 public/images/ 目录
2. 检查路径是否为 /images/ 开头
